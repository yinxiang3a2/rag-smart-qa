## 一、项目概述

### 1.1 项目背景

RAG（检索增强生成）智能问答系统是一个基于企业私有知识库的智能对话平台，支持文档上传、智能问答、记忆管理等核心功能。系统采用前后端分离架构，后端基于 FastAPI 构建 RESTful API，前端基于 Streamlit 提供交互界面。

### 1.2 核心能力

| 功能模块 | 描述 |
|---------|------|
| 文档管理 | 支持 PDF/Word/图片上传，自动分块与向量化 |
| 智能问答 | 基于 RAG 的混合检索问答，支持 BM25F + FAISS + RRF |
| 记忆系统 | 分层记忆管理（会话语料→短期索引→每日日志→长期画像） |
| 技能插件 | 支持内置技能（天气查询）和用户自定义技能扩展 |
| 会话管理 | 多会话支持，对话历史持久化 |

**检索策略**：
- BM25F权重：content=1.0, filename=0.5
- RRF常数：K=60
- 混合配额：60%向量优先 + 双命中文档优先
- 相似度阈值：0.3（30%）

## 三、核心模块设计

### 3.1 RAG 服务 (rag_service.py)

**双层架构**：
- **调度层**：意图分析 + 技能匹配 + 参数提取
- **回答层**：上下文构建 + LLM生成

**关键流程**：
```python
def query(question, session_id):
    # 1. 调度层路由
    route = router(question)
    
    # 2. 技能执行（如需）
    if route.need_skill:
        skill_result = execute_skill(route.skill_type)
    
    # 3. 混合检索
    chunks = vector_store.hybrid_search(query, use_bm25f=True)
    
    # 4. 记忆检索（仅个人信息查询）
    if should_use_memory(question):
        memory_context = memory_search(question)
    
    # 5. LLM生成
    answer = llm.generate(question, context)
    
    # 6. 自动记忆记录
    auto_log_conversation()
```

### 3.2 向量存储 (vector_store.py)

**存储结构**：
```python
{
    "_chunk_meta": [{chunk_id, doc_id, content, created_at}, ...],
    "_doc_to_indices": {doc_id: [faiss_index, ...], ...},
    "_doc_meta": {doc_id: {filename, ...}, ...},
    "_deleted_docs": set()  # 软删除标记
}
```

**检索方法**：
- `search()`: 纯向量检索
- `_bm25f_search()`: 纯关键词检索
- `hybrid_search()`: BM25F + FAISS + RRF 混合检索

### 3.3 记忆服务 (memory_service.py)

**检索算法**：BM25F + RRF 混合检索
- 字段权重：key=2.0, tags=1.5, content=1.0
- 融合策略：RRF(K=60)

**自动压缩机制**：
- 每10条消息触发一次LLM提炼
- 从每日日志提取关键信息
- 更新短期记忆索引

## 四、API 接口设计

### 4.1 聊天接口

```
POST /api/v1/chat
Request: { "question": string, "session_id": string?, "top_k": int? }
Response: { "answer": string, "sources": [], "session_id": string }
```

### 4.2 文档接口

```
GET  /api/v1/documents              # 文档列表
GET  /api/v1/documents/{doc_id}     # 文档详情（包含所有chunks）
DELETE /api/v1/documents/{doc_id}  # 删除文档
POST /api/v1/upload               # 上传文档
```

### 4.3 记忆接口

```
GET  /api/v1/memory/stats           # 记忆统计
GET  /api/v1/memory/corpus          # 会话语料列表
GET  /api/v1/memory/index/stats    # 索引统计
GET  /api/v1/memory/long-term      # 长期记忆
POST /api/v1/memory/extract         # 手动提取长期记忆
```

### 4.4 技能接口

```
GET  /api/v1/skills/               # 技能列表
POST /api/v1/skills/reload         # 重载技能
GET  /api/v1/skills/template/default # 默认模板
```

## 九、技术风险与对策

| 风险 | 影响 | 对策 |
|-----|-----|------|
| MiniMax API限流 | 问答响应慢 | 指数退避重试+本地缓存 |
| 大文档处理超时 | 上传失败 | 分片上传+异步处理 |
| 向量库规模膨胀 | 检索性能下降 | 定期压缩+分区 |
| 记忆系统数据冗余 | 存储浪费 | 自动清理+压缩策略 |
